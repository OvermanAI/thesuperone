> STAGE 2 開啟 — 你已是老闆。現在，蓋一座替你工作的工廠
> 警告：這一章要先用一門鞋生意，徹底改變你對「工廠」的想像 _

Boss 的身分你拿到了，Boss Engineering 的技能也開始練了。

現在動土，蓋你的第一座 AI 工廠。但在搬第一塊磚之前，先用一門最簡單的生意，把「工廠」這件事想清楚。

---

## CH04-1｜先講一門運動鞋生意

你想做一門運動鞋生意。你有三條路。

**第一條：自己動手做。** 花三年學會做鞋，然後一天做一雙。你的產量＝你的雙手 × 工時。做得再拚，天花板就是「一個你」。停手，就停產。

**第二條：請一個老師傅幫你做。** 還是一天一雙，而且做多少由他決定，你受制於他。你沒打破天花板——只是把它從「你的雙手」換成「別人的雙手」，還多了一份不可控。

**第三條：蓋一家鞋廠。** 機器 24 小時自動量產，產量不再綁在任何一個人的雙手上。

每個做生意的人心裡都清楚：**第三條才是答案。** 因為做生意要的是「擁有產能」，不是「親手生產」。前兩條路再怎麼努力，本質都是用自己的時間換錢。

---

## CH04-2｜為什麼「工廠」感覺是大公司的事——以及為什麼 AI 工廠不是

那為什麼不是每個人都去蓋工廠？因為實體工廠貴到嚇人：**廠房**（地、租金）、**大量員工**（薪水、管理）、**生產設備**（機器、產線）。這些都要先砸大筆資本。

所以自古以來，「蓋工廠」就是大公司的特權。**這就是為什麼你一聽到「AI 工廠」，直覺也覺得那是大公司在玩的。** 你的直覺沒錯——對「實體工廠」完全正確。

但 AI 工廠，把實體工廠那三道資本門檻全部拿掉了：

```
廠房      → 不用，跑在雲端
大量員工  → 不用，Agent 員工按需生成、用完即停、不領薪水
生產設備  → 不用，你的「設備」就是 Claude Code + Agent
```

**AI 工廠唯一的成本，是 API token——用多少、付多少。** 沒有「先砸一筆才能開工」的固定資本，只有跟著產量走的變動成本。

所以你可以**從零、一個人**開始蓋，按需擴大規模。這是人類史上第一次，一個人也可以擁有一座工廠。

## CH04-3｜你的第一座工廠：最小組合

別想一千個 Agent。你的第一座工廠，小到只有一台機器。

從你**現在已經在做的事**出發——你大概已經會用 Claude Code 幫你寫程式了吧？只要往前推一步：

```
① 現狀（多數人在這）
   你 ──→ Claude Code（直接幫你寫程式，你逐一審每段產出）

② 你的第一座工廠 v0
   你(AI Boss) ──→ Claude Code(AI CEO) ──→ 一個 Coding Agent
   你給目標和標準，Claude Code 指派 Coding Agent 寫程式，
   並負責「讓它的成果達到你的標準」

③ 之後（AI Big Boss）
   把這套放大到 10 倍、百倍規模
```

跨過 ②，你就成為 **AI 全端創業家**——能從零打造與運營一座 AI 工廠的人。

**為什麼第一座工廠先用「寫程式」當例子？** 因為程式有確定性檢查（跑測試、跑 lint 就知道對不對）→ 你的「標準」天然可驗證 → AI CEO 真的扛得起品質 → **這座工廠今天就跑得起來。** 這是最容易先成功的起點。（內容、設計這種「標準主觀」的產線更難，我們後面專門談。）

---

## CH04-4｜核心原則：CEO 不是你

①和②看起來只差一層，但那一層是整個關鍵：

> 在①，**你**是那個檢查每段程式碼對不對的人。
> 在②，**Claude Code（CEO）**才是檢查 Coding Agent 成果的人——你只定標準、審 CEO。

把「成果品質的監督權」交給 AI CEO，這就是 **AI 工廠的 CEO 不是你**的真正意思。也是為什麼「光走到 ② 就很少人做得到」——誠實的三個原因：

1. **不敢放手**：多數人不信任，繼續自己逐行檢查 → 永遠卡在①。
2. **給不出可驗證標準**：沒把「好」變成 CEO 能檢查的東西，CEO 就扛不起品質（CH03 的 Boss Engineering 命門）。
3. **要付 token 與編排紀律**：兩層比一層貴，要管調度。

跨過這三關，你才真正「擁有一台會自己跑的機器」，而不只是「一個手很快的工人」。

## CH04-5｜放大＝AI Big Boss：為什麼不能「一個 Claude Code 直連一千個」

很多人以為放大很簡單：把一千個 Agent 直接掛到一個 Claude Code 上就好。

**這在現在的能力下一定爆掉。** 原因是一個硬限制：**context window（上下文窗口）**——它是 agent 系統裡最稀缺的資源。一千個 Agent 的工具說明、回傳、推理痕跡，全擠進同一個窗口，十分鐘就塞爆，然後它開始忘記前文、品質崩，或瘋狂吞 token、帳單爆。

所以放大不是「加 Agent」，是**換結構**——這正是 AI Big Boss 那一階的功夫：

```
你 → AI CEO（協調者，只調度，不把一千段對話塞自己窗口）
       ↓ 分派
   中間管理層（sub-orchestrators／範圍明確的工作流）
       ↓ 各自啟動
   短命 sub-agents（每個有「自己獨立的 context window」，做完即丟）
```

讓它不爆的三機制：**context 隔離**（每個員工自己的窗口）、**外部持久狀態**（進度放外部資料庫，不塞對話裡）、**context 管理**（摘要／淘汰／分階段載入）。

好消息：這跟你的 v0 是**同一個形狀**，只是員工從 1 變多。所以從 v0 學會編排，放大就是「加員工＋加中間層」，不是重學。

## CH04-6｜誠實的成本與難度

我必須講其他 AI 創業書不敢講的：**這座工廠不是免費，也不是一鍵。**

- **token 會燒**：分派越多員工 × 越長的 context × 重試，token 消耗越快。成本控制本身是技能：context 隔離、model routing（便宜的模型做便宜的事）、能寫死就寫死。
- **大半其實是寫死的程式碼**：不是每個節點都丟給 LLM。能用腳本／規則／API 做的就寫死——更便宜、更可靠、不幻覺。只有「需要判斷」的節點才呼叫 Agent。「一千個 Agent 的工廠」真實樣貌＝少數協調者 ＋ 大量確定性管線 ＋ 在判斷點才啟動的短命 agent。
- **連 v0 都很難**：我蓋了一條內容產線跑三週，它確實 24 小時在產，但產出還是次級品——不是機器壞了，是品檢標準沒設好。**工廠會跑，不代表自動產好貨。** 這正是 Boss Engineering 的戰場。

這不是要嚇退你。相對於請一百個人、砸千萬資本，這仍然是作弊碼——只是門票是 token ＋ 學會當廠長，不是免費。

## CH04-7｜今天的第一步：蓋你的第一台機器

別想整座工廠。今天只蓋**一台會跑的機器**：

1. 挑一個**有確定性檢查**的小任務（最適合：一個你會寫的小程式／工具／自動化腳本，跑得起來就知道對不對）。
2. 用 CH03 的 Boss Engineering 寫好目標 + 標準（標準＝它要通過哪些檢查）。
3. 交給 Claude Code 當 AI CEO：「你指派一個 agent 去做，並負責讓成果通過這些檢查，再回報給我。」
4. **關鍵練習：忍住不要自己跳下去改。** 讓 CEO 去管那個員工。你只看「有沒有達標」。

跑通這一台，你就從「用 Claude Code 的人」，正式變成「擁有一座 AI 工廠的 AI 全端創業家」——哪怕現在只有一台機器。

---

你蓋出了第一台機器。但一台只會「做」的機器還不夠——它要能驗證自己的成果、還要真的有人買單。
