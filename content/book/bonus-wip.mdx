> 你手上這本書，就是用這套方法寫出來的。現在，整套送你 _

你讀到的《The Super One》，不是寫完才出版的。它是**一章一章、公開地、在網站上長出來的**——連每一次「改了什麼」都攤在 [建造日誌](/log) 上。

這套方法，我叫它 **WRITING IN PUBLIC（WIP）**。這份贈品，把它完整交給你，讓你也能用 **Claude Code + Obsidian + 一個電子書網站**，公開寫出你自己的書。

---

## 一、什麼是 WRITING IN PUBLIC

矽谷有個概念叫 **Building in Public**——公開地建造你的產品與事業，把過程攤給世界看。

WIP 是我把它用在「寫書」上的版本：

> **WRITING IN PUBLIC = 把一本書的「寫作過程」本身，變成公開、可迭代、會長大的內容。**

不是寫完一本完美的書才上架，而是：今天寫一章、今天就上線；收到回饋、明天就改；連修訂紀錄都公開。讀者看到的不是成品，是**一本活書**。

---

## 二、為什麼 WIP 勝過「寫完再出版」

傳統寫書：關起門寫一年 → 出版 → 才知道有沒有人要。漫長、孤獨、零回饋、風險全壓在最後。

WIP 把這條線整個翻轉：

```
寫完再出版          WRITING IN PUBLIC
──────────────────────────────────────
關門寫一年       →   邊寫邊上線
出版才有回饋     →   每章上線就收回饋
過程是隱形的     →   過程本身就是內容與行銷
押注一次         →   一章一章驗證、迭代
```

- **邊寫邊驗證**：哪章有人讀、有人留言，市場一路告訴你。
- **過程即內容**：你的寫作旅程，本身就是吸引讀者的「創辦人實境秀」。
- **複利**：每上線一章，都是一個會持續被搜尋、被分享的資產。

---

## 三、WIP 的三件套

整套只需要三個工具，一個人就能跑：

| 工具 | 角色 | 一句話 |
|---|---|---|
| **Obsidian** | 真稿來源 | 你在這寫、在這改。唯一的真相來源。 |
| **Claude Code** | AI 寫作搭檔 + 工程師 | 幫你搭站、轉檔、上架、自動化——你不用會寫程式。 |
| **電子書網站** | 公開發布平台 | Next.js + MDX 靜態站 + Vercel，`git push` 自動部署。 |

**關鍵**：你只在 Obsidian 寫作。建站、上架、部署這些「工程」，全部交給 Claude Code——這正是本書 Boss Engineering 的精神：你當老闆，工程交給 AI。

---

## 四、WIP 寫作迴圈

每寫或改一章，就跑這個迴圈：

```
① 在 Obsidian 寫 / 改一章
        ↓ 跟 Claude Code 說「上架這章，這次改了…」
② Claude Code：清語法 + 補 frontmatter + 寫入網站 + git push
        ↓ 自動
③ Vercel 部署 → 章節上線（約 1-2 分鐘）
        ↓
④ 讀者讀到、留言、給回饋
        ↓
⑤ 回 Obsidian 改 → 每次改都留下一筆「修訂紀錄」（日期 + 變更內容）
        ↺ 回到 ①
```

你只做 ① 和 ⑤（寫作與判斷）。②③ 全是 Claude Code 的事。

---

## 五、WIP 的五條原則（從實戰長出來的）

1. **真稿在 Obsidian，網站是出口（單向同步）。** 永遠在 Obsidian 改；網站的檔由 Claude Code 產生，不手動碰。反向不同步——直接改網站會被下次發布覆蓋。
2. **狀態驅動的活書。** 每章標 `規劃中 / 草稿中 / 已發布`。讀者看得到全書骨架，章節一格格亮起來——這種「正在長大」的張力，是 WIP 的魅力。
3. **修訂紀錄：連「改了什麼」都公開。** 每次改不直接覆寫，而是追加一筆「日期 + 變更內容」，彙整成公開的[建造日誌](/log)。**這是 WIP 比 Building in Public 更進一步的地方**——迭代的痕跡本身就是內容。
4. **誠實準則。** 草稿就標草稿，不假裝完成；不虛構數字與成績。誠實本身，就是公開寫作最強的信任資產。
5. **每章一個「帶得走的工具」。** 模板、清單、提示詞、可驗證標準——讓讀者讀完不只「懂了」，而是「能用了」。

---

## 六、怎麼從零搭起你的 WIP（交給 Claude Code）

你不需要會寫程式。打開 Claude Code，照這個順序請它做（詳細提示詞見[贈品二](/book/bonus-prompts)）：

```
1. 建一個 Obsidian vault 當你的書房（真稿）
2. 請 Claude Code 搭一個 Next.js + MDX + Tailwind 靜態書站
   （首頁 / 目錄 / 章節頁 / 建造日誌）
3. 推上 GitHub，接 Vercel 自動部署，綁上你的網域
4. 請 Claude Code 做一支「發布腳本」：Obsidian → 網站 + 追加修訂紀錄
5. 開始寫第一章 → 上架 → 公開
```

整個搭建，一個下午，一個人，零後端、近乎零成本（只付網域與 token）。
