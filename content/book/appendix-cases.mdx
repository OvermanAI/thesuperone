## CH01 實戰案例：把「一條腿」重新想像成三位一體

**情境**
阿哲有 4 萬追蹤，靠健身短影音接業配，收入卡在「有接才有錢、停更就歸零」。他從沒想過自己是一門可以三條腿走路的事業——他只覺得自己是個「做影片的人」。

**最佳實務做法**
先不要急著做產品。第一步是把腦中模糊的「我做健身內容」，攤成一張清楚的三位一體藍圖：你的品牌（被記住什麼）、媒體（已有什麼流量）、電商（能賣什麼）。用 Claude Code 當戰略顧問逼出這張圖。

**關鍵 Claude Code 提示詞**
```md
你是我的 OPC 戰略顧問。我目前是單一事業的一人公司，我要把它
重新想像成「品牌 × 媒體 × 電商」三位一體的 Super OPC。

我的現況：
- 我做的內容主題：健身 / 自我成長短影音
- 我的受眾：想變強但不知從何開始的年輕男生
- 我目前的收入來源：業配
- 我已有的資產：4 萬追蹤、約 200 支影片

請幫我做三件事：
1. 診斷我目前最強和最弱的是哪一條腿（品牌 / 媒體 / 電商），說明理由。
2. 幫我各畫出三條腿「半年內可達成」的第一版樣貌（具體、可執行）。
3. 指出三條腿要怎麼互相導流，接成一個飛輪。

一次只先問我需要補充的關鍵資訊，再給藍圖。
```

**你會得到**
一張「我的三位一體藍圖 v1」：清楚看到自己媒體最強、品牌中等、電商是 0，以及第一條該補的腿——這就是後面所有章節的施工順序。

## CH02 實戰案例：三力地基健檢，鎖定最弱的一塊

**情境**
阿哲知道要蓋工廠了，但他只會剪片和寫貼文。商業（怎麼找到有人付錢的題目）、其他創作力（編程／音樂）、把媒體變成事業——他不確定自己缺哪一塊、該先補哪裡。

**最佳實務做法**
用 Claude Code 跑一次「三力地基健檢」，把模糊的不安變成一張有優先序的補強清單。先量化現況，再只補「最痛、最擋路」的那一塊，不要三塊一起貪。

**關鍵 Claude Code 提示詞**
```md
你是我的能力教練。請幫我為「AI 全端創業家的三力地基」做健檢。
三力是：
A. 商業力（AI 精實創業：能找到有人願意付錢的題目）
B. 全方位 AI 創作力（編程 / 圖文 / 音樂 / 影片）
C. 一人品牌（已在運行的媒體或電商）

請依序一題一題問我（一次一題），問完後：
1. 幫每一項用 1–10 評分，並說明依據。
2. 指出哪一塊是我現在「最擋路」的瓶頸（不是最弱，是最該先補的）。
3. 給我一份 30 天補強計畫，只聚焦那一塊，每週一個具體可驗證的小目標。
```

**你會得到**
一份個人化健檢：阿哲發現自己媒體（C）很強、創作力（B）只會影片缺其他、商業力（A）幾乎是 0——瓶頸是「從沒驗證過有人願意付錢」。30 天計畫於是聚焦 A，而不是急著學寫程式。

## CH03 實戰案例：把 Claude Code 正式任命為你的 AI CEO

**情境**
阿哲過去把 AI 當「問問題的工具」——想到才用、用完就忘。他要做的轉變，是把最強的 Agent 當成替他運營的 AI CEO，並開始建立創辦人知識庫，讓它用「阿哲的方式」做事。

**最佳實務做法**
兩個動作：(1) 改變下指令的方式——從「幫我寫一段文案」升級成「這個結果交給你，你來拆解、執行、回報」；(2) 開一份創辦人知識庫，先寫一頁「我是誰」，餵給 AI CEO。

**關鍵 Claude Code 提示詞**
提示詞一｜任命 AI CEO、用「結果導向」下任務：
```md
從現在起，你是我的 AI CEO，負責運營我的一人公司。
我的角色是 Big Boss：我只出願景、給方向、做最後審核；
你負責把目標拆解成任務、執行、必要時調度子任務，最後向我回報。

第一個任務（我給結果，不給步驟）：
「我要在本週內，產出一個能測試『付費健身計畫』需求的最小方案。」

請你：
1. 先把這個目標拆解成你打算執行的步驟與產出清單。
2. 標出哪些需要我做決定 / 提供素材。
3. 等我確認後再開始執行，過程中遇到分叉點主動問我。
```

提示詞二｜開立創辦人知識庫（第一頁）：
```md
幫我建立一份「創辦人知識庫」的起始檔，用來讓你（AI CEO）和所有
Agent 都按「我」的方式工作。請用訪談的方式，一次問我一題，
引導我把以下三件事講清楚，最後整理成一份結構化文件：

1. 我的主張：我相信什麼、反對什麼（健身 / 自我成長領域）
2. 我的語氣：我會說 / 不會說的句子各 3 個
3. 我的標準：我絕不妥協的原則

整理時請保留我的原話風格，不要改寫成官腔。
```

**你會得到**
一個被正式任命、會「先拆解再執行」的 AI CEO，加上一份創辦人知識庫 v1——往後工廠的所有產出，都從這份知識庫長出阿哲的味道。

## CH04 實戰案例：給 AI CEO 一份營運章程，讓你只管一個人

**情境**
阿哲任命了 AI CEO，但他還是忍不住每個產出都跳下去改——又快變回那個被工作埋住的人。他要做的，是把「介面」從一對多收斂成一對一：他只對 AI CEO，AI CEO 去管其他所有 Agent。

**最佳實務做法**
寫一份「AI CEO 營運章程」——把你對品質、流程、何時該問你、何時自己決定，一次講清楚，存進創辦人知識庫。從此你不必每次重講，AI CEO 照章程自主運營整條產線。

**關鍵 Claude Code 提示詞**
```md
你是我的 AI CEO。我要把運營介面收斂成「我只管你一個」。
請和我一起草擬一份《AI CEO 營運章程》，內容涵蓋：

1. 決策邊界：哪些事你可以自己決定、哪些必須先問我（請你先提案，我再調整）
2. 品質標準：產出在送到我面前之前，必須先通過哪些自我檢查
3. 回報節奏：你多久、用什麼格式跟我回報（我只想看重點與待決策）
4. 調度原則：當一個任務需要多個 Agent，你如何拆分與分派

請先針對「健身內容產線」提出第一版章程草案，
再問我三個最關鍵、會影響章程的問題。最後輸出成可存檔的文件。
```

**你會得到**
一份《AI CEO 營運章程》。阿哲之後只需審核 AI CEO 的回報，不再逐一改成果——工廠開始長大，他的負擔卻沒變。

## CH05 實戰案例：先賣再做——一頁銷售頁驗證需求

**情境**
阿哲想做一套「30 天從零開始健身」的付費計畫。舊本能是先花一個月把課做到完美。新打法：先用一頁銷售頁，確認真的有陌生人願意付錢，再動手。

**最佳實務做法**
先 SELL：做一頁清楚說明「賣什麼、賣多少」的銷售頁 + 一個收訂金／早鳥的連結，丟到受眾面前，看陌生人是否掏錢。有綠燈，才進 BUILD。

**關鍵 Claude Code 提示詞**
提示詞一｜SELL：產出驗證用銷售頁：
```md
你是我的 AI CEO。我要驗證一個產品需求：「30 天從零開始健身計畫」。
請依創辦人知識庫的語氣，幫我做一頁「預售驗證銷售頁」，包含：

1. 一個打中痛點的標題
2. 這計畫解決什麼、給誰、不給誰
3. 內容大綱（先寫得讓人想買，但不需我先做完）
4. 早鳥定價與一句「為什麼現在就要買」
5. 一個明確的行動呼籲（預購 / 訂金）

目標是「測試有沒有人願意付錢」，不是上線完整產品。
另外給我 3 個我該觀察的訊號，判斷這個需求是否成立。
```

提示詞二｜BUILD（拿到綠燈後才用）：
```md
預售已驗證有人付錢。現在把「30 天健身計畫」正式 BUILD 出來。
請你拆解成 Agent 工作流：內容生產、交付方式、客服問答。
先給我產線設計與你打算交給哪些 Agent，我確認後開始執行。
```

**你會得到**
一頁能立刻測試需求的銷售頁，以及一個明確的 Go / No-Go 判準。阿哲用一頁的力氣，避開了「做完才發現沒人要」的幾個月白工。

## CH06 實戰案例：每週讓 AI CEO 改系統，不改成果

**情境**
阿哲的內容產線開始跑了，但成效時好時壞。他過去的反應是「這篇爛，我自己重寫」。這章他要學會：把問題交給 AI CEO，改的是「產生成果的規則」，不是單次成果。

**最佳實務做法**
建立一個每週迭代迴圈：AI CEO 掃上週數據 → 找出規律 → 把規律寫回工作流規則 → 下週全線照新規則跑。你只看它要改哪條規則、批准與否。

**關鍵 Claude Code 提示詞**
```md
你是我的 AI CEO。我們來跑這週的迭代優化，原則是「改系統，不改成果」。

請依序執行：
1. 量測：盤點上週內容產線的數據（曝光 / 互動 / 點擊到銷售頁），
   告訴我哪些表現最好、最差。
2. 學習：歸納出「為什麼好的好、差的差」——找出可複製的規律。
3. 調整：把這些規律，寫成可以加進我們內容工作流的「新規則」
   （例如格式、開頭、發布時間的準則），而不是只重寫某幾篇。
4. 列出你建議新增 / 修改的規則清單，標注預期影響，等我批准。

我只想做一個動作：看你的規則建議，然後說 yes / 調整。
```

**你會得到**
一條會自我升級的產線：阿哲每週只花 10 分鐘審規則，工廠的內容品質卻一週比一週穩——而且這些改善是寫進系統的，不是靠他一次次補救。

## CH07 實戰案例：設計品牌產品階梯，先讓工廠做出第一階

**情境**
阿哲只有業配收入，沒有自己的產品。他要把「健身教練阿哲」這個品牌，Build 成一排由低到高、讓粉絲一階一階往上買的產品。

**最佳實務做法**
先設計四階階梯（免費內容 → 低價工具包 → 中價課程 → 高價服務），再挑最容易做的低價那一階，交給工廠先生產出來，快速跑通「品牌→產品」這條路。

**關鍵 Claude Code 提示詞**
提示詞一｜設計階梯：
```md
你是我的 AI CEO。依創辦人知識庫，幫我設計「健身教練阿哲」的品牌產品階梯。
請給出四階，每階都要具體（不要只寫類別）：

- 免費（曝光）：用哪種內容當入口
- 低價（第一筆交易）：什麼模板 / 工具包 / 電子書，定價區間
- 中價（主力）：什麼課程 / 社群
- 高價（頂端）：什麼服務 / 一對一 / 共同建造

每一階註明：賣給誰、解決什麼、如何自然把人帶往上一階。
```

提示詞二｜生產第一階：
```md
我們先做低價那一階：「居家徒手健身 30 動作圖解工具包」。
請你調度 Agent 把它做出來：內容（動作說明）、版面設計、
一頁銷售說明，全部照我的語氣。先給我產出計畫與分工，我確認後執行。
```

**你會得到**
一張完整的品牌產品階梯藍圖，加上第一個真的能賣的低價產品——阿哲第一次有了「不靠業配、自己擁有」的收入入口。

## CH08 實戰案例：一個真實片段，量產成一週內容

**情境**
阿哲知道「真實」才是 AI 取代不了的內容，但他一個人沒辦法天天產出大量貼文、影片。他要做的，是只負責「真實地記」，把量產交給工廠。

**最佳實務做法**
每天記下一個「真實片段」（一個決定、一次失敗、一個數字），交給 AI CEO，依創辦人知識庫的語氣，量產成跨平台、跨格式的一週內容。你出真實，工廠出數量。

**關鍵 Claude Code 提示詞**
```md
你是我的 AI CEO，負責我的「創辦人實境秀」內容線。
這是我今天的真實片段（5 句話）：

「我把『30 天計畫』的銷售頁標題改了，從強調效果改成強調『不用上健身房』，
預購點擊明顯變多。原來我的受眾最怕的不是沒效果，是麻煩。」

請依我的語氣，把這個真實片段量產成一週、跨平台內容：
1. 一篇深度貼文（拆解我學到什麼，能幫到同樣在做產品的人）
2. 一支短影音腳本（前後對比 + 一個洞察）
3. 一張數據 / 觀點圖卡的文案
4. 電子報裡的一段

全部聽起來都要像「我」，不要像通用 AI 內容。
```

**你會得到**
一個 5 分鐘的真實片段，變成一週、橫跨多平台的內容。阿哲的媒體線開始用「真實的他」量產曝光，而不是被演算法埋掉的通用內容。

## CH09 實戰案例：在自有通路接上自動成交

**情境**
阿哲過去靠平台私訊談業配、用別人的購物車賣東西，客戶資料都不是他的。他要建一條自己的成交路徑：在自己的通路上，把信任自動接成成交。

**最佳實務做法**
把「銷售頁 → 結帳 → 沒買的自動跟進 → 交付」整條路徑，變成 AI CEO 顧著的 Agent 工作流，跑在你自己的網站／通路上，24/7 自動收錢、自動留下客戶資料。

**關鍵 Claude Code 提示詞**
```md
你是我的 AI CEO。我要把「居家徒手健身工具包」的成交，
搬到我自己的通路上，並且自動化。請幫我設計並列出可執行步驟：

1. 著陸與信任：把媒體流量導到我自有頁面，頁面上放哪些見證 / 內容建立信任
2. 銷售頁 → 結帳：怎麼接上金流，讓付款順到沒有摩擦
3. 自動跟進：對「看了沒買」的人，設計 2–3 封自動信的內容與節奏
4. 交付 + 客服：付款後自動送達、常見問題由 Agent 回覆
5. 客戶資料：怎麼把每個買家留進我自己的名單，方便日後再行銷

先給我整條路徑的設計與你要交給哪些 Agent，我確認後再執行。
```

**你會得到**
一條跑在自有通路、24/7 自動成交的路徑，而且每個買家都進了阿哲自己的名單——他第一次「擁有客戶」，而不是租用平台的客戶。

## CH10 實戰案例：接上第一條飛輪線，讓三條腿互相加速

**情境**
阿哲的品牌、媒體、電商都各自能跑了，但還是三門獨立的小生意。他要把它們接成一個會互相加速的飛輪——讓一條腿的產出，自動變成下一條腿的燃料。

**最佳實務做法**
先接最強兩條腿之間的一個導流接點（阿哲媒體最強，先接媒體→電商），讓它自動化。跑順一條，再一條條補其餘接點，並盤點底下正在堆疊的四種資產。

**關鍵 Claude Code 提示詞**
```md
你是我的 AI CEO。我要開始把三條腿接成飛輪。先接「媒體 → 電商」。

請幫我設計並自動化這個導流：
1. 在每一則內容（貼文 / 影片 / 電子報）的適當位置，
   自然地把人導向我的產品階梯——不要硬推，要符合我的語氣。
2. 設計一套規則，讓 Agent 依內容主題，自動選擇導向最相關的那一階產品。
3. 告訴我這條接通後，會回頭餵養哪些資產（內容 / 品牌 / 客戶 / 系統），
   以及下一個最該接的飛輪接點是哪一條。

先給我設計與規則，我確認後讓你接進現有工作流。
```

**你會得到**
第一條接通的飛輪線：阿哲的每一則內容都自動把人帶向產品，收入又回頭養更多內容。他清楚看到四種資產開始堆疊——生意從「加法」轉成「指數」。

## CH11 實戰案例：挖深那道只有你有的護城河

**情境**
阿哲擔心：等這套打法普及，別人也蓋一樣的工廠怎麼辦？答案是——對手抄得走工廠，抄不走「阿哲」。他要親手挖深唯一靠他自己的護城河：他的故事與主張。

**最佳實務做法**
其餘三道護城河（三線互鎖、自動變寬、勞動力不對稱）會隨工廠自然形成；唯一要親手挖的是第一道。持續把「只有你經歷過的故事、最不一樣的主張」寫進創辦人知識庫，讓品牌深到無法被複製。

**關鍵 Claude Code 提示詞**
```md
你是我的 AI CEO，也是我的品牌策略顧問。
我要挖深「只有我有」的護城河——我這個人本身。

請用訪談方式，一次問我一題，挖掘以下素材（問完幫我整理進創辦人知識庫）：
1. 一個只有我經歷過、和健身 / 自我成長有關的真實轉折故事
2. 一個我和市場上其他健身內容「最不一樣」的主張
3. 一個我絕不妥協、就算少賺也要堅持的原則

整理後，請再幫我做一件事：
盤點我目前四道護城河（你=品牌 / 三線互鎖 / 自動變寬 / 一人勝百人）
各自的強弱，指出我接下來最該補強哪一道。
```

**你會得到**
更深的創辦人知識庫，以及一份護城河健檢。阿哲的品牌開始長出「對手就算全抄做法也複製不了」的那一層——因為那一層是他本人。

## CH12 實戰案例：定義你的新活法與公開建造計畫

**情境**
工廠跑起來、收入進來後，阿哲面對最後一個問題：然後呢？他要從「賺錢的創作者」，升級成一個活出新活法、並公開帶路的新世代領導者。

**最佳實務做法**
這一步不靠工具，靠想清楚。用 Claude Code 幫你把「你要用這套系統創造什麼、過什麼樣的人生、帶誰一起走」講清楚，並把它變成一個公開建造的 90 天計畫。

**關鍵 Claude Code 提示詞**
```md
你是我的人生與品牌教練。我已經建好了我的 AI 工廠與三位一體事業。
現在幫我想清楚「然後呢」——我要成為什麼樣的新世代領導者。

請用提問引導我（一次一題），釐清三件事：
1. 我把時間買回來之後，最想拿來創造什麼、過什麼樣的生活
2. 我想用我的公開建造，幫哪一群人、帶他們走哪一條路
3. 我希望別人想到我時，記住的是什麼

最後幫我輸出：
- 一句話的「我的新活法」宣言
- 一個 90 天的公開建造計畫：我要公開記錄什麼、產出什麼，
  讓我的存在本身成為別人敢於改變的理由
```

**你會得到**
一句屬於你自己的新活法宣言，加上一份 90 天公開建造計畫。阿哲不再只是個高收入創作者——他開始活成一張別人可以跟隨的地圖，成為新世代領導者。
